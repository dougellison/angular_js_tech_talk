What is it?

  • 100% JavaScript
  • Client Side
  • Templating System
  • Lightweight < 29KB compressed and minified
  • Yes it is a framework
  • Model View Whatever...
  • Tons of Utility Methods
  • Uses JQuery if available

Why use it?

  • Client Side means responsive
  • Its Comprehensive
    • Built in support for RESTFul operations
    • Vast templating possibilities
    • Testable
    • Many Utility methods
  • Dependency Injection
  • Very large active community http://angular-ui.github.io/
  • Lots of people use it http://builtwith.angularjs.org/
  • 2 Way data binding
  • Create custom components

How does it compare to BackboneJS


Below are excerpts directly from the BackboneJS website http://backbonejs.org/#faq

  • View agnostic
  • Goal is to be a library
  • Avoids Two way data-binding
  • No support for making new HTML tags or entities
  • HTML and Logic are completely separated
